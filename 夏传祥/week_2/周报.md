## 二、核心学习内容

### 2.1 Java SE 收尾

- **网络编程**：学习了 Socket 通信模型，理解了 TCP 三次握手与四次挥手的基本流程，动手实现了简单的 C/S 聊天程序，初步了解了 BIO 阻塞式 IO 的工作方式。

- **反射机制**：掌握了 Class 对象的获取方式（Class.forName、类名.class、getClass），理解了反射在框架底层中的核心地位。重点练习了通过反射获取构造方法、成员变量和成员方法，并能绕过访问权限调用私有成员。

- **注解**：学习了元注解（@Target、@Retention 等）的含义与用法，理解了注解的声明周期（SOURCE → CLASS → RUNTIME），能够自定义简单注解并通过反射解析。

### 2.2 Java Web 入门

- **HTML 基础**：
    - 掌握了 HTML 文档基本结构（`<!DOCTYPE>`、`<html>`、`<head>`、`<body>`）
    - 学习了常用标签：标题（h1~h6）、段落（p）、链接（a）、图片（img）、列表（ul/ol/li）、表格（table/tr/td/th）
    - 理解了块级元素与行内元素的区别，以及 div 和 span 的使用场景
    - 学习了表单标签（form、input、select、textarea）及其常用属性，能独立编写简单的注册/登录表单页面
    - 了解了语义化标签（header、nav、main、section、footer）的意义

- **开发工具**：配置了 Maven 环境，了解了 Maven 的坐标体系和依赖管理机制，能够在 pom.xml 中声明依赖并成功导入。

## 四、问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| 反射调用私有方法时抛出 `IllegalAccessException` | 在调用前使用 `setAccessible(true)` 绕过访问权限检查 |
| Socket 通信中客户端断开后服务端抛出异常导致程序终止 | 在服务端循环中加入 try-catch，捕获异常后继续等待下一个连接，而不是直接退出 |
| HTML 表单 `GET` 与 `POST` 的使用场景分不清 | 查阅资料后总结：GET 用于数据查询/检索，参数暴露在 URL 中，有长度限制；POST 用于数据提交/修改，参数放在请求体中，安全性更高 |
| Maven 依赖下载缓慢 | 将 Maven 仓库镜像切换为阿里云镜像，下载速度显著提升 |

## 六、总结与思考

本周顺利完成了从 Java SE 到 Java Web 的过渡，是整个学习路线上的一个重要节点。回顾这两周，有以下几点体会：

- **基础扎实才能走得更远**：学习反射和注解时，很多概念都依赖于集合、泛型等前置知识。第一周打下的基础在本周发挥了明显的支撑作用，这让我更加确信"先精后广"的学习策略是有效的。

- **先理解再动手，效率更高**：Socket 编程涉及网络协议、IO 流等多方面知识，如果直接上手写代码容易迷失。本周先花时间理清了通信模型和数据流向，再动手实践，差错时间明显缩短。

- **前端不可绕过**：原以为前端只是辅助，实际学习后发现，不理解 HTML/CSS/JS 就很难理解前后端如何协作，也无法独立完成一个完整的 Web 项目。接下来的学习会在此基础上稳步推进。

整体进度符合预期，下周将继续保持当前节奏，重点攻克 Servlet 核心内容。
