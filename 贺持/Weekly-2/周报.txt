一、Unity脚本组件
	1、组件定义：Unity中任何游戏物体均无任何功能，如果想要为其添加功能，就需要为它添加相应功能的组件，而每一个组件其实就是一个引擎内部的组件脚本或是由自己编写的组件脚本。

	2、脚本的生命周期；

方法名称			调用时间
Awake			最早调用，所以一般可以在此实现单例模式
OnEnable			组件激活后调用，在 Awake 后会调用一次
Start	在 Update 	之前调用一次，在 OnEnable 之后调用，可以在此设置一些初始值
FixedUpdate		固定频率调用方法，每次调用与上次调用的时间间隔相同
Update			帧率调用方法，每帧调用一次，每次调用与上次调用的时间间隔不相同
LateUpdate		在 Update 每调用完一次后，紧跟着调用一次
OnDisable			与 OnEnable 相反，组件未激活时调用
OnDestroy		被销毁后调用一次


using System.Collections;
using System.Collections.Generic;
using UnityEngine;
public class Test : MonoBehaviour
{
    private void Awake()
    {
        Debug.Log("awake");
    }
    //组件使用被调用
    private void OnEnable()
    {
        Debug.Log("OnEnable");
    }
    // Start is called before the first frame update
    void Start()
    {
    }
    //每一帧都会调用
    void Update()
    {
    }
    //会紧跟Update( )执行完后跟着执行
    private void LateUpdate()
    {  
    }
    //每隔固定间隔调用一次
    private void FixedUpdate() {
        //Debug.Log("FixedUpdate");
    }
    //组件取消被调用
    private void OnDisable()
    {
        Debug.Log("OnDisable");
    }
    //组件销毁才会调用
    private void OnDestroy()
    {
        Debug.Log("OnDestroy");
    }
}



	3、脚本运行顺序：系统随机分配脚本组件权重，由低到高顺次执行所有脚本的Awake( )方法，再执行	所有脚本的OnEnter( )方法......

	自定义脚本执行顺序：Edit--->Script Execution Order




二、Unity旋转：欧拉角，四元数
	1、定义：
	欧拉角：用三个独立的角度（x、y、z）表示绕对应轴的旋转，单位是度（°）。 在 Unity 里，transform.eulerAngles 就是物体的欧拉角旋转值。

	四元数：用四个浮点数（x、y、z、w）表示旋转，本质是复数的扩展，数学上基于三维旋转的“轴角”表示法。在 Unity 里，transform.rotation 就是物体的四元数旋转值。

	2、优点与缺点
	欧拉角：
	优点：直接用三个数值就能控制旋转，适合做简单的角度控制（比如人物朝向、物体固定角度）。

	缺点：万向节死锁（Gimbal Lock）：当两个旋转轴重合时，会丢失一个		   自由度，导致旋转行为异常。
	
      	不适合做连续旋转、平滑插值，比如相机跟随、物体旋转动画
	很容易出现奇怪的抖动。

	四元数：
	优点：彻底避免万向节死锁，适合所有连续旋转、平滑插值场景。 
	旋转计算效率高，能直接做乘法实现旋转叠加

	缺点：不直观，直接修改 x/y/z/w 很难手动控制角度，一般不直接写数值。

	3、二者间互相转换
	欧拉角--->四元数
	// 绕Y轴旋转90度的四元数 
	Quaternion rot = Quaternion.Euler(0, 90, 0); 
	// 给物体设置旋转 
	transform.rotation = rot;

	四元数--->欧拉角
	Quaternion rot = transform.rotation; 
	Vector3 euler = rot.eulerAngles; // 得到对应的欧拉角(x,y,z) 
	Debug.Log(euler.y); // 查看Y轴的旋转角度
	


三、Unity预制体与变体
	1、预制体
	定义：预制体是 Unity 中用于复用 GameObject 及其组件、属性的模板资源，存于 Project 窗口，可反复拖入场景生成实例。 	
	作用：统一管理与批量更新。改预制体，所有场景实例自动同步，实例上的手动修改会保留。 
	创建方式：Hierarchy 中做好 GameObject → 拖入 Project 窗口。 
	实例：场景中拖出来的副本，默认关联预制体；可做覆盖（Override），脱离预制体部分属性控制。 
	编辑模式：双击预制体进入Prefab Mode，独立编辑不影响场景。

	2、预制体变体
	定义：变体是基于基础预制体（Base Prefab）派生的特殊预制体，继承基础并可覆盖、增删内容，基础改则变体同步，变体改不影响基础。
	使用场景：做 “同根不同形” 的系列对象（如不同血量、皮肤的敌人、同车型不同配色），避免重复预制体，改基础即可批量更新所有变体。
	
	3、变体规则
	继承优先：未覆盖属性跟随基础；变体覆盖->基础预制体Unity。 	
	可嵌套：变体可作为其他变体的基础，形成层级Unity。 
	覆盖管理：Inspector 顶部Overrides可查看：revert、apply 
	覆盖；Apply All to Base会把变体修改强写回基础。

四、Unity Vector3
	1、本质：一个包含 x、y、z 三个 float 分量的结构体
	2、用途：
	作为坐标：transform.position = new Vector3(1, 2, 3); 表示物体在世界空间的位置 
	作为向量：transform.forward * speed 表示物体沿自身前方的移动方向和速度

	常量				含义						值
	Vector3.zero		零向量或原点				(0, 0, 0)
	Vector3.one		全 1 向量					(1, 1, 1)
	Vector3.right		世界空间的 X 轴正方向（右）	(1, 0, 0)
	Vector3.left		世界空间的 X 轴负方向（左）	(-1, 0, 0)
	Vector3.up		世界空间的 Y 轴正方向（上）	(0, 1, 0)
	Vector3.down		世界空间的 Y 轴负方向（下）	(0, -1, 0)
	Vector3.forward	世界空间的 Z 轴正方向（前）	(0, 0, 1)
	Vector3.back		世界空间的 Z 轴负方向（后）	(0, 0, -1)
	
	3、常用成员方法：
	Vector3.Angle(a, b)：计算两个向量之间的夹角（单位：度），范围 [0, 180] 
	Vector3.Distance(a, b)：计算两个坐标点之间的直线距离（三维空间的欧氏距离） 
	Vector3.Magnitude：向量的模长（长度），new Vector3(1,0,1).magnitude 就是根号2
	Vector3.Normalize()：将向量单位化（变成长度为 1 的方向向量，保留方向） 
	Vector3.Lerp(a, b, t)：线性插值，在两个向量之间按比例 t（0~1）做平滑过渡

	4、基础运算
	加减：对应分量相加减，常用于位置偏移（比如 transform.position + Vector3.right * 2） 
	乘除：和标量相乘、相除，常用于缩放向量长度（比如 Vector3.forward * 5 表示向前移动 5 个单位） 
	注意：向量不能直接和其他向量相乘，要用 Vector3.Dot（点积）或 Vector3.Cross（叉积）做向量运算

	5、常见使用场景
	物体移动：transform.Translate(Vector3.forward * speed * Time.deltaTime); 
	方向判断：用 Vector3.Angle 判断敌人是否在你的前方视野里 
	距离检测：用 Vector3.Distance 判断玩家和敌人的距离是否小于攻击范围 
	平滑移动：用 Vector3.Lerp 做相机跟随、物体移动的平滑过渡
